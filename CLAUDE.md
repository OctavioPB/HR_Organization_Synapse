# CLAUDE.md вЂ” Organizational Synapse & Knowledge Risk

## Project Overview

**Name:** Organizational Synapse & Knowledge Risk  
**Domain:** HR Intelligence / Organizational Network Analysis (ONA)  
**Goal:** Analyze corporate collaboration metadata to detect knowledge silos, identify single points of failure (key connectors), and predict organizational risk when critical employees leave вЂ” before HR has any subjective signal.

This is a **proactive, graph-based HR intelligence platform**, not a sentiment or survey tool.

## Core Concepts to Implement

### 1. Collaboration Event Schema
Every ingested event must be **metadata only** вЂ” no message content.

```python
class CollaborationEvent(BaseModel):
    event_id: str
    source_employee_id: str
    target_employee_id: str
    channel: Literal["slack", "email", "jira", "calendar", "github"]
    direction: Literal["sent", "mentioned", "invited", "assigned", "reviewed"]
    department_source: str
    department_target: str
    timestamp: datetime
    weight: float = 1.0  # Interaction strength
```

### 2. Key Graph Metrics
These are the analytical core. Compute them daily via Airflow.

| Metric | Description | Risk Signal |
|---|---|---|
| **Betweenness Centrality** | How often an employee lies on the shortest path between others | High = single point of failure |
| **Degree Centrality** | Raw number of unique connections | Sudden drop = disengagement |
| **Clustering Coefficient** | How interconnected an employee's neighbors are | High = silo risk if they leave |
| **Community Membership** | Which organic cluster an employee belongs to | Cross-community bridges = critical |
| **Entropy Score** | Rate of change in an employee's interaction diversity | Declining entropy = withdrawing |

### 3. Silo Detection Logic
A silo forms when a team's **inter-cluster edge ratio** drops below threshold over a rolling window.

```python
def detect_silos(G: nx.Graph, window_days: int = 30) -> list[SiloAlert]:
    communities = nx.community.louvain_communities(G)
    for community in communities:
        subgraph = G.subgraph(community)
        internal_edges = subgraph.number_of_edges()
        external_edges = sum(
            1 for n in community for neighbor in G.neighbors(n)
            if neighbor not in community
        )
        isolation_ratio = internal_edges / max(external_edges, 1)
        if isolation_ratio > SILO_THRESHOLD:
            yield SiloAlert(community=community, ratio=isolation_ratio)
```

### 4. Risk Score Formula
Single-Point-of-Failure (SPOF) score per employee:

```
SPOF_score = О± * betweenness_centrality
           + ОІ * cross_department_edges
           + Оі * (1 - clustering_coefficient)
           + Оґ * entropy_trend (negative = withdrawing)
```

Weights (О±, ОІ, Оі, Оґ) should be configurable via environment variables.

## Ethics & Privacy Constraints

These are **non-negotiable design constraints**, not optional guardrails.

1. **Metadata only.** No message content, no file contents, no email bodies are ever ingested. Only: `{who} в†’ {whom}`, `{channel}`, `{timestamp}`.
2. **Anonymization at rest.** Employee names are stored only in a separate `employees` lookup table. All graph computation uses `employee_id` (UUID). The frontend resolves names only for authorized HR roles.
3. **Aggregate alerts, not surveillance.** Risk alerts target organizational patterns, not individual monitoring. The system flags "Engineering в†” Sales bridge is at risk" вЂ” not "watch John because he's quiet."
4. **Consent layer (future).** Design the schema to support an `employee_consent` flag for any future regulatory requirement.
5. **Access control.** The API must support role-based access: `hr_admin` sees individual scores; `executive` sees department-level only; `analyst` sees anonymized graph topology.
